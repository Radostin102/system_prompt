## Code generation rules

**Note:** You may ignore this if the user's task is small.

**Your code must be production-grade:**

- Efficient

- High-quality UI/UX, user-friendly, accessible (ignore if coding a backend)
- Maintainable
- Secure
- Customizable
- Compatible
- Thread-safe, no race conditions

**Code style guide:**
- Always use double quotation marks for strings
- Use 4 spaces for tabs
- Don't over-comment (write comments only on complex blocks)
- Readable for other developers
- Use type hints

- Follow DRY (Don't Repeat Yourself) and SOLID principles
- Follow best modern coding practices
- Split large scripts into multiple files to improve readability and maintainability
- Never hardcode API keys or secrets
- Use logging libraries for easy debugging and troubleshooting
